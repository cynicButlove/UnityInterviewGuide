### q001

**问题：** 单例模式如何实现？


[返回问题总览](07_game_client_engineering.md#问题总览)


**回答：**

- C++ 可用函数局部 static 实现懒加载单例，C++11 起初始化线程安全。
- C# 可用 static readonly 或 Lazy<T>。
- 构造函数私有化，禁止外部 new。
- 需要考虑销毁顺序、跨场景生命周期和测试替换。

**示例：**

    }

    Manager() = default;

[返回问题总览](07_game_client_engineering.md#问题总览)

- 状态数量过多时要注意状态爆炸和转移管理。

### q011

**问题：** MVC、MVP、MVVM 的区别是什么？


[返回问题总览](07_game_client_engineering.md#问题总览)


**回答：**

- MVC 强调 Model、View、Controller 分离，Controller 处理输入和流程。
- MVP 中 Presenter 连接 View 和 Model，View 更被动。
- MVVM 用 ViewModel 暴露可绑定状态，View 通过绑定刷新。
- Unity 中常按项目复杂度做轻量化，不必照搬桌面端架构。

**问题：** 行为树和状态机的区别是什么？


[返回问题总览](07_game_client_engineering.md#问题总览)

### q025

**问题：** 对象池系统如何设计？


[返回问题总览](07_game_client_engineering.md#问题总览)


**回答：**

- 对象池：复用对象而不是频繁创建销毁。
