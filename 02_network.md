## TCP / UDP

### q001

**问题：** TCP 和 UDP 的区别是什么？


[返回问题总览](02_network.md#问题总览)


**回答：**

- TCP 面向连接、可靠、有序、字节流；UDP 无连接、尽力而为、保留报文边界。
- TCP 适合可靠传输、登录、聊天、下载等；UDP 适合实时状态同步、语音、动作游戏。
- TCP 有重传和拥塞控制，但会带来队头阻塞。
- UDP 需要应用层补可靠、乱序、重复、拥塞和安全机制。

### q016

**问题：** TCP 为什么会出现粘包？


[返回问题总览](02_network.md#问题总览)


**回答：**

- TCP 是字节流协议，没有业务消息边界。
- 发送端 Nagle、接收端缓冲、应用读取大小不同都会让数据合并或拆分。

## Socket 与网络编程

### q021

**问题：** Socket 是什么？


[返回问题总览](02_network.md#问题总览)


**回答：**

- Socket：应用层使用传输层能力的编程接口。
- 关键点：通常由协议、IP、端口描述一个通信端点。

### q057

**问题：** 如何设计 ACK 和重传？


[返回问题总览](02_network.md#问题总览)


**回答：**

- 每个可靠包分配递增序号并放入发送缓存。
- ACK 可确认最大连续序号，也可用 bitmask 确认最近若干包。
- 超时未确认时重传，RTO 可根据 RTT 动态调整。
- 重传次数和缓存大小要有限制，避免雪崩。

[返回问题总览](02_network.md#问题总览)
