## 进程与线程

### q001

**问题：** 进程和线程的区别是什么？

[返回问题总览](03_operating_system.md#问题总览)

**回答：**

- 进程是资源分配单位，线程是 CPU 调度单位。
- 进程有独立地址空间，线程共享进程地址空间。
- 进程切换和通信开销通常更大。
- 线程共享数据方便但需要同步。
- 对比题要先给结论，再按语义、底层机制、性能成本和适用场景展开。
- 本题的核心比较维度是资源隔离、调度、上下文切换、并发模型和任务拆分，不要只停留在名字相似或 API 形式不同。
- 进程是资源隔离单位，线程是调度执行单位。
- 线程共享地址空间，通信方便但更容易出现数据竞争。
- 工程选择通常不是绝对优劣，要结合数据规模、调用频率、可维护性和错误成本。
- 客户端常把资源加载、解压、网络收包、日志写入放到工作线程，主线程保持帧率稳定。
- 线程不是越多越好；共享内存带来效率，也带来竞态、死锁和生命周期问题。
- 线程的定义：CPU 调度执行的基本单位，同进程线程共享地址空间。

- 追问方向：让你在真实项目里二选一，并解释取舍依据。
- 回答时主动补充“什么时候不适合用”，会比只说优点更像有经验。
- 落地检查：可用线程堆栈、Profiler Timeline、任务队列统计和上下文切换指标验证。

- 优化/排查题第一步永远是定位瓶颈，先拿数据再改代码。

- 小文件过多、随机 IO、主线程反序列化和同步解压都会造成加载卡顿。
- 修复优先级按收益/风险排序，先做低风险高收益项，再做结构性改造。
