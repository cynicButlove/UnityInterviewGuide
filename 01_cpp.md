### q001

**问题：** C++ 和 C 的区别是什么？

[返回问题总览](01_cpp.md#问题总览)

**回答：**

- C 偏过程式，C++ 在兼容 C 的基础上增加类、模板、异常、RAII 和标准库。
- C++ 有构造/析构和访问控制，能把资源生命周期封装到对象里。
- C++ 支持函数重载、引用、命名空间、泛型和运行时多态。
- 代价是语言规则和对象模型更复杂，ABI、异常和模板编译成本也更需要关注。
- 对比题要先给结论，再按语义、底层机制、性能成本和适用场景展开。
- 本题的核心比较维度是类型系统、对象模型、编译期规则和运行期行为，不要只停留在名字相似或 API 形式不同。
- 工程选择通常不是绝对优劣，要结合数据规模、调用频率、可维护性和错误成本。
- 在游戏客户端里，这类知识通常影响接口设计、资源所有权、模块边界和性能敏感代码。
- 不要只背语法名词，面试官更常追问它在对象生命周期、二进制布局或编译器行为上的影响。
- 收束时给出一个项目中的使用或排查场景，可以把答案从概念背诵提升到工程经验。

- 回答时主动补充“什么时候不适合用”，会比只说优点更像有经验。

[返回问题总览](01_cpp.md#问题总览)

**回答：**

- optional的核心作用是：表示一个值可能存在也可能不存在。
- 工程价值：避免用魔法值或额外 bool。
- 使用时注意：访问前应检查 has_value。
- 这类题的关键不是把名词堆出来，而是说明每一项解决的问题和限制。
- 这些选项通常可以按现代 C++ 类型推导、移动语义、并发库和类型工具分类理解。
- 主流方案要优先说明，特殊方案再补充适用条件和代价。
- 面试官常追问每一项的适用场景、性能成本和失败模式。
- 现代 C++ 能减少手写资源管理，但错误使用 move、lambda 捕获和 atomic 会制造隐蔽 bug。
- 要区分语法糖和真实语义，例如 std::move 不移动、auto 可能丢引用、atomic 不保护复合不变量。
- 收束时给出一个项目中的使用或排查场景，可以把答案从概念背诵提升到工程经验。

**面试补充：**

- 追问方向：底层实现、性能影响、典型使用场景和常见错误。
- 如果能结合项目中的一次 bug、优化或设计取舍，回答会更有可信度。
- 落地检查：可用编译期类型检查、拷贝/移动日志和多线程测试验证。
