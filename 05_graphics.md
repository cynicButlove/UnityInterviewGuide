### q001

**问题：** 渲染管线的整体流程是什么？


[返回问题总览](05_graphics.md#问题总览)


**回答：**

- 应用阶段准备场景、剔除、排序和提交 DrawCall。
- 几何阶段进行顶点处理、坐标变换、裁剪等。
- 光栅化把图元转成片元。
- 片元阶段计算颜色并经过深度、模板、混合输出到帧缓冲。

### q002

**问题：** 应用阶段做了什么？


[返回问题总览](05_graphics.md#问题总览)


**回答：**

- 运行在 CPU 侧，处理游戏逻辑、动画、物理、剔除和渲染队列准备。
- 收集可见物体并设置材质、灯光、相机等状态。
- 向图形 API 提交 DrawCall。
- 很多渲染性能问题其实发生在应用阶段的提交成本。

### q008

**问题：** 顶点数据如何变成屏幕上的像素？


[返回问题总览](05_graphics.md#问题总览)


**回答：**

- 顶点先经过模型、视图、投影矩阵变到裁剪空间。

- 片元着色后通过深度/模板/混合写入帧缓冲。

### q016

**问题：** 深度测试和透明物体有什么关系？


[返回问题总览](05_graphics.md#问题总览)


**回答：**

- 透明物体通常开启 ZTest，避免被前方不透明物体错误显示。
- 透明物体通常关闭 ZWrite，避免挡住后续透明物体。
- 因为透明混合依赖顺序，所以还要从远到近排序。
- 复杂交叉透明需要特殊方案，如深度预写或 OIT。

### q018

**问题：** 正向渲染和延迟渲染的区别是什么？


[返回问题总览](05_graphics.md#问题总览)


**回答：**

- 正向渲染逐物体/逐光源计算，透明支持好，适合光源少或移动端。
- 延迟渲染先写 GBuffer，再统一计算光照，适合大量动态光。
- 延迟渲染显存和带宽更高，透明物体仍通常走正向。
- 移动端常因带宽和硬件限制偏正向或 Forward+。
