### q001

**问题：** 数组和链表的区别是什么？

[返回问题总览](06_data_structures_algorithms.md#问题总览)

**回答：**

- 数组连续内存，随机访问 O(1)，缓存友好。
- 链表节点分散，随机访问 O(n)，插入删除已知节点 O(1)。
- 数组扩容或中间插入成本高，链表额外指针开销大。
- 工程中若无强插删需求，数组/动态数组通常更快。
- 对比题要先给结论，再按语义、底层机制、性能成本和适用场景展开。
- 本题的核心比较维度是存储结构、复杂度、缓存局部性和工程适用场景，不要只停留在名字相似或 API 形式不同。
- 可以把每一项分别放到“发生时机、谁负责管理、是否有额外成本、错误后果”四个维度里解释。
- 工程选择通常不是绝对优劣，要结合数据规模、调用频率、可维护性和错误成本。
- 资源索引、事件表、排行榜、AOI、背包和缓存都离不开数据结构选型。
- 只背复杂度不够，还要考虑常数、内存布局、迭代器稳定性和并发访问。
- 收束时给出一个项目中的使用或排查场景，可以把答案从概念背诵提升到工程经验。
